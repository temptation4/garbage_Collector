# ♻️ Java Garbage Collector

Java developers often use objects every day, but few understand what happens to them after they're no longer needed.

This repository is dedicated to understanding **how Java manages memory automatically**, how the JVM performs Garbage Collection, and how different GC algorithms impact application performance.

Whether you're preparing for Java interviews or trying to optimize your applications, this repository provides practical explanations instead of just theory.

## What you'll learn

- ♻️ JVM Memory Model
- 🧠 Heap vs Stack
- 🌱 Young, Old & Metaspace
- 🚀 Serial, Parallel, G1, ZGC & Shenandoah GC
- ⚡ GC Logs & Monitoring
- 📊 Memory Leaks & OutOfMemoryError
- 🎯 Java Interview Questions
